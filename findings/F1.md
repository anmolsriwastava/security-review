# Plaintext Password in User Details API

**Severity:** Critical
**Endpoint:** POST /user/details

## Description

The /user/details endpoint returns the user's password in the response.
The password is visible in plaintext.

I tested this with two different passwords. I changed my password and the
new password also appeared in the response. This means the server is not
hashing passwords properly. If passwords were hashed the server could not
show the new one back to me.

## Steps to Reproduce

1. Log in to Acadza
2. Open DevTools > Network tab > Fetch/XHR
3. Navigate to dashboard or profile so the /user/details request fires
4. Click the request and check the Response tab
5. The password field contains the actual password in plaintext
6. Change the password and repeat steps 1 to 5. The new password also
   appears in plaintext

## Impact

Anyone who can read this API response gets the real password. This includes
browser extensions, anyone on the same network, or if a malicious script
ever runs on the page.

The bigger issue is that passwords appear to be stored in a way that the
server can read them back. If the database is ever breached every password
is exposed directly. Users who reuse passwords on other sites are at risk.
