# Exploration Log — 1

## What I Did Today

### Account Setup
Created a free trial account using phone number and OTP. Noticed right away
that the platform sets the phone number as both username and password. This
is not safe because phone numbers are easy to guess or find and a custom or more complex
password should be generated by default instead of normal phone number.

### API Mapping
Opened DevTools and watched network requests while using the platform. Found
these endpoints so far.

/user/details — POST request. Needs JWT token. Returns everything about the
user including phone, email, purchased chapters, plan type, and password.

/user/login — POST request. No auth needed. Takes phone and password and
returns a JWT token if credentials are correct.

/coupongoldplan/list/usercoupon — GET request. Needs auth. Returns 404
right now so maybe this feature is not active yet.

### JWT Token
Decoded my auth token at jwt.io. It uses HS256 algorithm and contains my
user ID, role as student, and expiry date about 20 days from now.

### IDOR Testing
Tried changing the userid in /user/details request body to another user's
ID. The server returned my own data not theirs. So IDOR does not work on
this endpoint. The server is using the token to identify me not the
parameter I send. Good security here.

### Video Content Protection
Checked how paid video lectures are protected. Videos are hosted on Vimeo
with signed CDN URLs. The Vimeo player has domain restriction so the video
cannot be played on other websites. CDN URLs have an expiry time. A free
user cannot access the video page at all.

### Unauthenticated Access
Tried accessing API endpoints directly without logging in. All endpoints
returned Auth failed message. Authentication is enforced properly.

## Findings are mentioned in findings folder. 

Plaintext Password in API : 
The /user/details endpoint returns the password in the response. Changed
my password and the new one also appeared. This means passwords are not
hashed properly. If database leaks all passwords are exposed.

CORS Wildcard : 
API responses have Access-Control-Allow-Origin set to star. Any website
can make requests to the API.

Server Info in Headers : 
Response headers show nginx version and Express. Helps attackers know
what software is running.
